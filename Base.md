# プロジェクト概要
スマートフォンで手軽に遊べる「10を作れ！（Make 10）」のWebアプリを開発します。
モダンなフロントエンド環境で構築し、ブラウザからホーム画面に追加して遊べる**PWA（Progressive Web Apps）**として実装してください。

# 技術スタック
- **ビルドツール**: Vite
- **フレームワーク**: React
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS (または標準のCSS Modules。モダンでレスポンシブな実装ができるもの)
- **PWA化**: `vite-plugin-pwa` プラグインを使用

# 画面構成（モバイルファースト）
画面は縦向き（ポートレート）のスマートフォンを前提に配置してください。

1. **ヘッダー**: アプリタイトル「Make 10」と、現在の「連勝数（スコア）」。
2. **ディスプレイ領域**: ユーザーが入力した計算式が大きく表示されるエリア。
3. **数字ボタン領域**: 今回与えられた4つの数字ボタン（2×2等）。一度タップするとDisabledになる状態管理を行うこと。
4. **演算子ボタン領域**: 「＋」「－」「×」「÷」と、「(」「)」の括弧ボタン。
5. **コントロールボタン領域**:
    - 「1文字消す (BS)」ボタン
    - 「クリア (C)」ボタン（入力をリセットし、数字ボタンのDisabledを解除）
    - 「判定 (Enter)」ボタン

# コア機能とゲームロジック要件
ロジックはReactのカスタムフック等に分離し、テスト・保守がしやすい設計にしてください。

## 1. 問題生成ロジック
- 0〜9の数字から4つをランダムに選出する。
- **【重要】必ず「10が作れる組み合わせ」のみを出題すること。**（事前に正解パターン配列を用意しておくか、生成時に探索アルゴリズムでチェックしてください）。

## 2. 入力制御（状態管理）
- `useState` 等を用いて、ユーザーの入力式（文字列や配列）と、各数字ボタンの使用状況（boolean）を管理する。
- 4つの数字は「必ず1回ずつ」すべて使い切らなければならない。

## 3. 計算と判定ロジック
- 「判定」ボタン押下時のチェック：
  1. 4つの数字がすべて使われているか？
  2. 数式として文法的に正しいか？（安全な評価関数を実装するか、`new Function`で×を*に、÷を/に変換して計算）
  3. 計算結果が「10」と一致するか？
- 正解時：スコアのStateを+1し、祝賀UIを表示後、次の問題へ遷移。
- 不正解時：エラーUIを表示し、現在の入力Stateをリセット。

# PWA実装要件 (vite.config.ts)
- `vite-plugin-pwa` を導入し、`vite.config.ts` で以下の設定を行うこと。
  - `registerType: 'autoUpdate'`
  - `manifest`: name("Make 10 Puzzle"), short_name("Make 10"), theme_color, background_color, icons設定。
  - オフライン対応のため、静的アセットをキャッシュする設定を含めること。

# UI/UX（デザイン）の要件
- タップしやすいようボタンは大きめ（最小44x44px）に設定。
- ネイティブアプリの操作感に近づけるため、CSSで `user-select: none;` や `touch-action: manipulation;` を適用し、意図しないテキスト選択やズームを防ぐこと。
