# PRD: Make 10 -- アニメーション強化 & ギブアップ機能

## 変更履歴

| 版 | 日付 | 内容 |
|----|------|------|
| v1 | - | UI リデザイン（モダン & ポップ & 子ども向け）-- 実装済み |
| v2 | 2026-02-28 | アニメーション強化 + ギブアップ（答え参照）機能の追加 |

---

## 1. 問題定義

### v1 で解決済みの課題

UI リデザインにより、明るくポップな配色、グラスモーフィズム、ボタンの視認性向上、正解時のフィードバック強化は完了している。基本的なアニメーション（数字ボタンのポップイン、スコアのバウンス、紙吹雪エフェクト、ボタンタップのスケール変化）も実装済み。

### v2 で取り組む課題

1. アニメーションの物足りなさ
   - 現在のアニメーションはボタン操作時と正解時に限定されており、操作していない時間帯（考え中、アイドル状態）に画面が静止している
   - 画面遷移（出題切り替え、フィードバック表示・非表示）がやや唐突で、滑らかさに欠ける
   - 正解時の演出は紙吹雪のみで、バリエーションが少ない
   - 子ども向けアプリとしては、画面全体がもっと「生きている」感覚が欲しい

2. 行き詰まり時の救済手段がない
   - 現在、解けない問題に出会うとスキップする手段がなく、同じ数字の前で手が止まる
   - 子どもが「分からない」状態で放置されると、フラストレーションが溜まり離脱につながる
   - 答えを見て「なるほど」と学ぶ体験がなく、学習としての深まりが弱い

## 2. ターゲットユーザー

（v1 から変更なし）

### ペルソナ A: 小学校低学年の子ども（7〜9歳）

- 足し算・引き算を学び始めた時期
- スマートフォンやタブレットに触り慣れている
- カラフルで動きのあるものに惹かれる
- 正解できたときの「やった!」感が遊び続けるモチベーション
- 分からないと飽きて別のアプリに移ってしまう

### ペルソナ B: 子どもに算数を楽しんでほしい保護者

- 子どもに渡して安心できるシンプルなアプリを求めている
- ゲームとしての楽しさと学習効果の両立を期待
- 広告や課金要素のない、PWA としてインストールできるものを好む
- 子どもが諦めずに続けられる仕組みがあると嬉しい

## 3. バリュープロポジション

v1 の「見た瞬間に遊びたくなる」体験に加え、v2 では2つの価値を追加する:

- 画面が常に楽しく動いている「見ていて飽きない」体験。背景のアンビエントアニメーション、式入力時の演出、正解時の多彩な祝福エフェクトにより、操作のどの瞬間も視覚的に気持ちいい。
- 「分からなくても大丈夫」という安心感。ギブアップボタンで答えの式を確認でき、解き方を学んだうえで次の問題に進める。行き詰まりが学びに変わる。

## 4. 機能要件（MoSCoW）

### 実装済み（v1 成果物）

以下はすべて実装完了しており、v2 でも維持する:

- M1〜M6: 配色、ボタン視認性、フィードバック強化、式表示改善、スコア表示強化、ゲームロジック維持
- S1: ボタンタップ時の active:scale アニメーション
- S3: 数字ボタンのポップインアニメーション（animate-pop-in）
- S4: Nunito フォントの導入

### Must Have（v2 で必須）

- M7: ギブアップ機能（答え参照 + 次の問題へ進む）
  - コントロールエリアに「ギブアップ」ボタン（またはアイコン）を配置する
  - タップすると確認ダイアログを表示する（誤タップ防止）
  - 確認後、その問題の正解式を1つ画面に表示する
  - 答え表示中はキーパッド操作を無効にする
  - 答えを確認した後、タップで次の問題に進む
  - ギブアップした問題ではスコアは加算しない
  - ギブアップボタンは式が空の状態でも押せる（考える前にスキップしたいケースに対応）

- M8: 答えの生成ロジック
  - 与えられた4つの数字に対して、10を作る有効な式を少なくとも1つ算出できる
  - ゲーム開始時（出題時）に答えを事前計算しておく
  - 表示する式は全角演算子（×, ÷）を使い、ユーザーが入力する式と同じ表記にする
  - 答えの式はなるべくシンプルなもの（括弧が少ない、演算子が基本的なもの）を優先する

- M9: 背景のアンビエントアニメーション
  - 背景にゆっくりと浮遊・移動する装飾要素（丸、星、図形など）を配置する
  - 操作を邪魔しない透明度と z-index で、前景コンテンツの背後に表示する
  - CSS アニメーションで実装し、JS ランタイムコストを最小にする
  - `prefers-reduced-motion: reduce` の場合はアニメーションを停止する

### Should Have（リリースまでに実装したい）

- S2: 画面遷移のアニメーション（v1 から継続）
  - 正解 / ギブアップ後の次の問題への切り替わりにスムーズなトランジションを入れる
  - フィードバックオーバーレイの表示・非表示のアニメーションを改善する

- S5: 式入力時のライブアニメーション
  - 数字や演算子を入力するたびに、式表示エリアの文字が軽くバウンスまたはフェードインする
  - 式が完成に近づく（4つの数字がすべて使われた状態）と、判定ボタン（=）がパルスする

- S6: 正解時の祝福演出バリエーション
  - 紙吹雪に加えて、スターバースト（星が放射状に飛ぶ）やキラキラエフェクトを追加する
  - 連続正解時には演出が段階的に豪華になる（ストリーク演出との連携）
  - 正解メッセージのテキストが数パターンからランダムで表示される

- S7: アイドルアニメーション
  - 一定時間操作がないとき、数字ボタンが軽くバウンスまたは揺れて「ここを押してね」と誘導する
  - ヘッダーの星マークが瞬く、式表示エリアのプレースホルダーが緩やかにフェードイン・アウトする

### Could Have（余裕があれば実装）

- C1: 連続正解のストリーク演出（v1 から継続）
  - 連続正解数を表示し、ストリーク中は背景やボタンの色が段階的に変化する

- C2: サウンドエフェクト（v1 から継続）
  - ボタンタップ音、正解音、不正解音、ギブアップ音を追加する
  - ミュートボタンを設ける

- C4: テーマ切り替え機能（v1 から継続）
  - 2〜3種類のカラーテーマを選べるようにする（例: パステル、ビビッド、宇宙）

- C5: ギブアップ時の答え表示演出
  - 答えの式を一文字ずつタイプライター風に表示する
  - 表示完了後に「なるほど!」といった学びを促すメッセージを添える

- C6: パーティクルエフェクト（入力追従）
  - ボタンタップ時にタップ位置から小さなパーティクルが弾けるエフェクト
  - 演算子入力時に異なるパーティクル（キラキラ、リング状など）

### Won't Have（今回のスコープ外）

- W1: ゲームモードの追加（タイムアタック、難易度選択など）
- W2: ユーザーアカウント / ログイン機能
- W3: ランキング / リーダーボード
- W4: チュートリアル / オンボーディング画面
- W5: 多言語対応
- W6: 広告や課金要素
- W7: ヒント機能（部分的な答えを段階的に開示する機能。ギブアップとは異なり、途中まで見せる仕組み）
- W8: ギブアップ回数の制限やペナルティ

## 5. デザイン方針

### カラーパレット

（v1 から変更なし。実装済みの配色を維持する。）

- 背景: 明るい暖色系グラデーション（amber-100, pink-100, indigo-100）
- 数字ボタン: 4色ポップカラー（rose, sky, emerald, amber）
- 演算子ボタン: violet-400
- コントロールボタン: slate-200, red-100
- 判定ボタン（=）: orange-to-pink グラデーション

### レイアウト

- 既存のレイアウト構造（ヘッダー / 式表示 / 数字パッド / 演算子パッド / コントロールパッド）は維持する
- ギブアップボタンはコントロールパッドの一部として配置する（レイアウト大幅変更なし）
- 背景アンビエント要素は position: fixed で配置し、既存コンテンツを覆わない

### ギブアップ UI のトーン

- ネガティブな表現（「あきらめる」「降参」）は避ける
- 「こたえを みる」「ヒントをもらう」のようなフレンドリーな文言にする
- 確認ダイアログも励ましのトーン（「こたえを みても いいかな?」）
- 答え表示画面では「こうやって とくんだね!」のような学びにつなげるメッセージ

### アニメーション原則

- 楽しさを生むが邪魔にならない: アンビエント要素は低い不透明度で緩やかに動く
- レスポンシブなフィードバック: ユーザー操作には即座に視覚的反応を返す（100ms 以内）
- 一貫したイージング: ease-out を基本とし、バウンス系は cubic-bezier で統一する
- 過度にしない: 画面上で同時に激しく動く要素は2つまでに制限する

## 6. 非機能要件

### パフォーマンス

- First Contentful Paint: 1.5 秒以内
- アニメーションは 60fps を維持する
- CSS アニメーション / Tailwind のトランジションを優先し、JS アニメーションライブラリの追加は最小限にする
- アンビエントアニメーションは will-change / transform のみを使い、レイアウト再計算を発生させない
- 追加する Web フォントは 1 ファミリーまでに制限する（Nunito 導入済み）
- 答え生成ロジックの計算は出題時に完了させ、ギブアップタップ時にはゼロ遅延で表示する

### アクセシビリティ

- ボタンの最小タップ領域: 48x48px
- テキストと背景のコントラスト比: WCAG AA 準拠（4.5:1 以上）
- 使用済みボタンの状態が色以外でも判別できる（透明度の変化 + disabled 属性）
- ギブアップボタンに適切な aria-label を付与する
- 答え表示は aria-live="polite" で読み上げ対応する
- `prefers-reduced-motion: reduce` ですべてのアニメーションを無効化する（既存実装を拡張）

### ブラウザ / デバイスサポート

- iOS Safari 16+
- Android Chrome 最新版
- 画面幅 320px〜428px を主な対象とする
- PWA としてのオフライン動作を維持する

### データ / プライバシー

- ローカルストレージのスコア保存の仕組みは変更しない
- 外部へのデータ送信は行わない
- Web フォントの読み込み以外に外部リクエストを追加しない

## 7. スコープ境界

v2 の変更が及ぶ範囲:

- src/components/ -- 新規コンポーネント追加（ギブアップ UI、アンビエント背景）、既存コンポーネントのアニメーション拡張
- src/hooks/useMake10.ts -- ギブアップ状態管理の追加（giveUp アクション、答えの保持）
- src/logic/ -- 答え生成ロジック（solver）の新規追加
- src/index.css -- アニメーション keyframes の追加
- src/App.tsx -- 新コンポーネントの組み込み

以下は明確にスコープ外とする:

- vite.config.ts の PWA 設定の変更
- ゲームルールの変更（4つの数字で10を作るルール自体）
- スコア計算ロジックの変更（正解 +1 のみ。ギブアップではスコア変動なし）
- 新しいページやルーティングの追加
- 外部ライブラリの大規模な追加

## 8. 成功指標

定性的な指標で判断する:

- 画面を眺めているだけでも楽しいと感じられること（アンビエントアニメーションの効果）
- 操作のたびに「気持ちいい」と感じられるレスポンスがあること
- 子どもが行き詰まったとき、ギブアップボタンを押して答えを確認し、学びを得てから次に進めること
- ギブアップが「恥ずかしい」ではなく「勉強になった」と感じられるトーンであること
- 既存の全テストがパスし、正解判定のロジックに変化がないこと
