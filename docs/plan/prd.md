## 1. 問題定義

### v1 で解決済みの課題

UI リデザインにより、明るくポップな配色、グラスモーフィズム、ボタンの視認性向上、正解時のフィードバック強化は完了している。基本的なアニメーション（数字ボタンのポップイン、スコアのバウンス、紙吹雪エフェクト、ボタンタップのスケール変化）も実装済み。

### v2 で解決済みの課題

1. アニメーションの物足りなさ
   - 背景のアンビエントアニメーション、式入力時の演出、正解時の多彩な祝福エフェクトにより改善済み

2. 行き詰まり時の救済手段がない
   - ギブアップ機能（答え参照 + 次の問題へ）により改善済み

### v3 で取り組む課題

初めて遊ぶ子どもに対する導入体験がない

- 現在、ゲーム開始直後から「4つの数字と演算子で式を組み立てる」フル操作を要求する。7歳の子どもにとって、白紙の状態から式全体を構築するのはハードルが高い
- 四則演算をまだ完全には理解していない段階の子どもは、最初の問題で「何をすればいいか分からない」まま止まってしまう
- 初回プレイで成功体験を得られないと、アプリを閉じて二度と開かない
- 段階的に難しくなる仕組みがないため、既に慣れた子にも新鮮さの変化がない
- 現在のスコア（星の数）は「何問正解したか」を示すだけで、成長実感やマイルストーンとしての意味がない

## 2. ターゲットユーザー

### ペルソナ A: 小学校低学年の子ども（7〜9歳）

- 足し算・引き算を学び始めた時期
- スマートフォンやタブレットに触り慣れている
- カラフルで動きのあるものに惹かれる
- 正解できたときの「やった!」感が遊び続けるモチベーション
- 分からないと飽きて別のアプリに移ってしまう
- (v3 追加) 最初はルールや操作方法がよく分からず、手が止まりやすい
- (v3 追加) 「自分でもできた!」という最初の成功体験が、継続プレイへの分岐点になる

### ペルソナ B: 子どもに算数を楽しんでほしい保護者

- 子どもに渡して安心できるシンプルなアプリを求めている
- ゲームとしての楽しさと学習効果の両立を期待
- 広告や課金要素のない、PWA としてインストールできるものを好む
- 子どもが諦めずに続けられる仕組みがあると嬉しい
- (v3 追加) 段階的に力がつく学習設計だと安心感がある
- (v3 追加) 子どもの成長が目に見える仕組み（レベルアップ）を喜ぶ

## 3. バリュープロポジション

v1 の「見た瞬間に遊びたくなる」体験と v2 の「分からなくても大丈夫」体験に加え、v3 では以下の価値を追加する:

- 初めての子どもでも「できた!」から始まる導入体験。最初は穴埋め形式で1箇所だけ答えればよく、ゲームのルールと成功体験を同時に学べる。星を集めるうちに自然とフル操作に到達する、スムーズなスキルの階段。

## 4. 機能要件（MoSCoW）

### 実装済み（v1 + v2 成果物）

以下はすべて実装完了しており、v3 でも維持する:

- M1〜M6: 配色、ボタン視認性、フィードバック強化、式表示改善、スコア表示強化、ゲームロジック維持
- M7: ギブアップ機能（答え参照 + 次の問題へ進む）
- M8: 答えの生成ロジック（solver による正解式の事前計算）
- M9: 背景のアンビエントアニメーション
- S1: ボタンタップ時の active:scale アニメーション
- S3: 数字ボタンのポップインアニメーション
- S4: Nunito フォント
- S2, S5, S6, S7: 画面遷移アニメーション、式入力ライブアニメーション、祝福演出バリエーション、アイドルアニメーション

### Must Have（v3 で必須）

- M10: 3段階の難易度レベル

  ゲームは以下の3段階で進行する。各レベルは星の獲得数によって自動的に昇格する。

  | レベル | 名前 | 星の範囲 | 内容 |
  |--------|------|----------|------|
  | 1 | 超かんたん | 0〜4 | 穴埋め形式: 完成済みの正解式の中から、演算子1つまたは数字1つが空欄になっている。ユーザーは空欄をタップして、選択肢から正しいものを選ぶ |
  | 2 | かんたん | 5〜9 | 部分組み立て形式: 4つの数字のうち2つは式の中に配置済みで、残り2つの数字と演算子をユーザーが埋める |
  | 3 | ふつう | 10〜 | 現在と同じフル操作: 4つの数字から式全体を自由に組み立てる |

- M11: 「超かんたん」モードの出題ロジック

  - solver が算出した正解式を1つ選び、その中の演算子1つまたは数字1つを空欄（ブランク）にする
  - 空欄の位置はランダムに決定する
  - 空欄に対する選択肢を3〜4個提示する（正解1つ + ダミー2〜3つ）
  - ダミーの選択肢は、見た目にもっともらしいが計算結果が10にならないものにする
  - ユーザーは空欄をタップして選択肢から選ぶだけで回答が完了する

- M12: 「かんたん」モードの出題ロジック

  - solver の正解式から、4つの数字のうち2つを式の中にあらかじめ配置した状態で出題する
  - 配置済みの2つの数字はボタンが「使用済み」状態で表示される
  - 残り2つの数字と必要な演算子はユーザーが自由に入力する
  - 判定ロジックはふつうモードと同じ（式全体が10になるかを評価する）

- M13: 難易度レベルの自動昇格

  - スコア（星）の累計に基づいて、レベルの切り替えが自動的に発生する
  - レベル昇格時にお祝い演出を表示する（「レベルアップ!」+ レベル名 + 短い祝福アニメーション）
  - 昇格は一方向のみ。レベルが下がることはない
  - スコアとレベルは localStorage に永続化する

- M14: 現在のレベル表示

  - ヘッダーエリアに現在の難易度レベル名を表示する（「超かんたん」「かんたん」「ふつう」）
  - 次のレベルまでの残りスター数がユーザーに伝わる表示にする（例: プログレスバー、「あと N もん!」テキスト）

### Should Have（リリースまでに実装したい）

- S8: レベルアップ演出

  - レベル昇格時に専用のオーバーレイを全画面で表示する
  - 新しいレベル名を大きく表示し、キラキラ / スターバーストで祝福する
  - タップで閉じると、新しいレベルの最初の問題が始まる
  - 子どもが「すごいことが起きた」と感じるインパクトのある演出にする

- S9: 「超かんたん」モードのガイド表示

  - 初回プレイ時（星 0 の状態）に、空欄をタップして選ぶことを示す簡単な視覚ガイドを表示する
  - テキストではなく、矢印やハイライトなどの視覚的な誘導とする
  - 1〜2問目だけ表示し、その後は自動的に消える

- S2: 画面遷移のアニメーション（v2 から継続）
  - 正解 / ギブアップ後の次の問題への切り替わりにスムーズなトランジションを入れる
  - フィードバックオーバーレイの表示・非表示のアニメーションを改善する

### Could Have（余裕があれば実装）

- C1: 連続正解のストリーク演出（v1 から継続）
  - 連続正解数を表示し、ストリーク中は背景やボタンの色が段階的に変化する

- C2: サウンドエフェクト（v1 から継続）
  - ボタンタップ音、正解音、不正解音、ギブアップ音、レベルアップ音を追加する
  - ミュートボタンを設ける

- C4: テーマ切り替え機能（v1 から継続）
  - 2〜3種類のカラーテーマを選べるようにする（例: パステル、ビビッド、宇宙）

- C5: ギブアップ時の答え表示演出
  - 答えの式を一文字ずつタイプライター風に表示する
  - 表示完了後に「なるほど!」といった学びを促すメッセージを添える

- C6: パーティクルエフェクト（入力追従）

- C7: 「超かんたん」で空欄の数を2つに増やすバリエーション

### Won't Have（今回のスコープ外）

- W2: ユーザーアカウント / ログイン機能

## 5. デザイン方針

### カラーパレット

（v1 から変更なし。実装済みの配色を維持する。）

- 背景: 明るい暖色系グラデーション（amber-100, pink-100, indigo-100）
- 数字ボタン: 4色ポップカラー（rose, sky, emerald, amber）
- 演算子ボタン: violet-400
- コントロールボタン: slate-200, red-100
- 判定ボタン（=）: orange-to-pink グラデーション

### レイアウト

- 既存のレイアウト構造（ヘッダー / 式表示 / 数字パッド / 演算子パッド / コントロールパッド）は維持する
- ギブアップボタンはコントロールパッドの一部として配置する（レイアウト大幅変更なし）
- 背景アンビエント要素は position: fixed で配置し、既存コンテンツを覆わない

### v3 追加: 難易度別の UI バリエーション

- 「超かんたん」モード
  - 式表示エリアに正解式を表示し、空欄部分をハイライトされたタップ可能なスロットとして表現する
  - 空欄スロットは点線の枠 + パルスアニメーションで「ここをタップしてね」と誘導する
  - スロットをタップすると、選択肢が下部にポップアップで表示される（現在のキーパッド位置を流用）
  - 選択肢は大きめのボタン（3〜4つ）で表示し、選びやすくする
  - 通常のキーパッド（数字パッド・演算子パッド）は非表示にする

- 「かんたん」モード
  - 式表示エリアには数字が2つ配置された状態のテンプレートを表示する
  - 配置済みの2つの数字は NumberPad 上で「使用済み」状態にする
  - 残りの操作は「ふつう」モードと同一のキーパッド UI で行う

- 「ふつう」モード
  - 現在の UI そのまま。変更なし

### レベル表示のトーン

- レベル名は子どもが読めるひらがな表記にする（「ちょうかんたん」「かんたん」「ふつう」）
- プログレス表示は前向きなトーン（「あと 3もんで レベルアップ!」）
- レベルアップ時の演出は派手に祝う（「やったね! レベルアップ!」）

### ギブアップ UI のトーン

（v2 から変更なし。）

- ネガティブな表現（「あきらめる」「降参」）は避ける
- 「こたえを みる」のようなフレンドリーな文言にする
- 確認ダイアログも励ましのトーン（「こたえを みても いいかな?」）
- 答え表示画面では「こうやって とくんだね!」のような学びにつなげるメッセージ

### アニメーション原則

（v2 から変更なし。v3 の新規アニメーションにも同じ原則を適用する。）

- 楽しさを生むが邪魔にならない: アンビエント要素は低い不透明度で緩やかに動く
- レスポンシブなフィードバック: ユーザー操作には即座に視覚的反応を返す（100ms 以内）
- 一貫したイージング: ease-out を基本とし、バウンス系は cubic-bezier で統一する
- 過度にしない: 画面上で同時に激しく動く要素は2つまでに制限する

## 6. 非機能要件

### パフォーマンス

- First Contentful Paint: 1.5 秒以内
- アニメーションは 60fps を維持する
- CSS アニメーション / Tailwind のトランジションを優先し、JS アニメーションライブラリの追加は最小限にする
- アンビエントアニメーションは will-change / transform のみを使い、レイアウト再計算を発生させない
- 追加する Web フォントは 1 ファミリーまでに制限する（Nunito 導入済み）
- 答え生成ロジックの計算は出題時に完了させ、ギブアップタップ時にはゼロ遅延で表示する
- (v3 追加) 穴埋め問題の生成（正解式からの空欄作成 + ダミー選択肢の生成）は出題時に完了させる

### アクセシビリティ

- ボタンの最小タップ領域: 48x48px
- テキストと背景のコントラスト比: WCAG AA 準拠（4.5:1 以上）
- 使用済みボタンの状態が色以外でも判別できる（透明度の変化 + disabled 属性）
- ギブアップボタンに適切な aria-label を付与する
- 答え表示は aria-live="polite" で読み上げ対応する
- `prefers-reduced-motion: reduce` ですべてのアニメーションを無効化する（既存実装を拡張）
- (v3 追加) 穴埋めの空欄スロットに適切な aria-label（「えらんでね」など）を付与する
- (v3 追加) 選択肢ボタンにも aria-label を付与する

### ブラウザ / デバイスサポート

- iOS Safari 16+
- Android Chrome 最新版
- 画面幅 320px〜428px を主な対象とする
- PWA としてのオフライン動作を維持する

### データ / プライバシー

- ローカルストレージにスコアとレベルを保存する（既存のスコア保存を拡張）
- 外部へのデータ送信は行わない
- Web フォントの読み込み以外に外部リクエストを追加しない

## 7. スコープ境界

v3 の変更が及ぶ範囲:

- src/hooks/useMake10.ts -- 難易度レベル状態の追加、レベル別の出題ロジック切り替え、レベルアップ判定
- src/logic/ -- 穴埋め問題生成ロジック（正解式からの空欄生成、ダミー選択肢生成）の新規追加
- src/components/ -- 穴埋め UI コンポーネント（空欄スロット、選択肢ボタン）の新規追加、Header のレベル表示拡張、レベルアップ演出コンポーネント
- src/App.tsx -- レベルに応じた UI の切り替え
- src/index.css -- レベルアップ演出用の keyframes 追加

以下は明確にスコープ外とする:

- vite.config.ts の PWA 設定の変更
- 4つの数字で10を作るゲームルール自体の変更
- solvableCombinations データセットの変更
- スコア加算ロジックの変更（正解 +1 のみ。ギブアップではスコア変動なし。レベルによるスコア倍率もなし）
- 新しいページやルーティングの追加
- 外部ライブラリの大規模な追加

## 8. 成功指標

定性的な指標で判断する:

- (v2) 画面を眺めているだけでも楽しいと感じられること
- (v2) 操作のたびに「気持ちいい」と感じられるレスポンスがあること
- (v2) ギブアップが「恥ずかしい」ではなく「勉強になった」と感じられるトーンであること
- (v3 追加) 初めてアプリを開いた子どもが、操作説明なしに最初の問題を正解できること
- (v3 追加) 穴埋め形式からフル操作への移行が自然で、子どもが「急に難しくなった」と感じないこと
- (v3 追加) レベルアップの瞬間に「やった!」と声を上げるような達成感があること
- (v3 追加) 保護者が子どもの画面を見たとき、レベルと進捗が一目で分かること
- 既存の全テストがパスし、「ふつう」モードの正解判定ロジックに変化がないこと
